Info

Introduction

This section provides a manual for the Confidence Interval visualization. It explains the meaning of the inputs, the correct specification of the input data table and known issues.

Inputs

  • Source Database: A dropdown menu to select the database or cluster of the table we want ot visualize (e.g., redshift-general). Once you make a secltion here, the Source Schema dropdown menu will be populated with all schemas within the selected database.
  • Source Schema: A dropdown menu to select the schema that contains the table to visualize. Again, making a selection here initiates a script to populate the Source Table dropdown menu with all tables within the selected schema.
  • Source Table: A dropdown menu to select the name of the table to visualize.
  • Get Table: Once the source database, schema and table have been selected, click this button to retrieve the column names of the selected table.

A short time after clicking the Get Table button a second set of toolbars should appear:

  • Property: A dropdown menu to select the column in the data table that contains values for the property of interest.
  • Group by: A dropdown menu to select the column in the data table that contains category labels. This menu is optional. If it is left blank, the plot will show a single confidence interval reflecting the entire data set, otherwise there will be a confidence interval for each unique category in the column selected from the Group by menu.
  • Lower bound: A numercial input that allows the user to specify the cutoff percentage of the lower tail end of the distribution. For example, for a 95% confidence interval, a typical choice for the lower bound would be 2.5.
  • Upper bound: A numercial input that allows the user to specify the cutoff percentage of the upper tail end of the distribution. For example, for a 95% confidence interval, a typical choice for the upper bound would be 97.5.
  • Draw Graph: Once you have selected the Property, Lower bound, Upper bound and optionally Group by variables, click this button to retrieve the necessary data and to start drawing the graph. This will also bring up a third and final set of dropdown menus that allows the user the sort the data according to the categoy names or the expected values.

Data table

To get the correct visualization, the data table needs to be organized as follows:

The plot will generate a bar for each row of the table. You are responsible for removing duplicate rows. You will need at least one column with labels, one column with mean values and one column each for the lower and upper bounds. The table can contain additional columns, the toolbar will let you select the columns that are relevant to the plot.

Known issues

The number of bars that can be plotted is limited (currently 277). Possible fix: make def_vis_height dependent on the number of rows in the input data set.
