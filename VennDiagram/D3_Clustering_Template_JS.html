Info

Introduction

This section provides a manual for the Recommender System visualization. It explains the meaning of the inputs, the correct specification of the input data table and known issues.

Inputs

  • Source Database: A dropdown menu to select the database or cluster of the table we want ot visualize (e.g., redshift-general). Once you make a secltion here, the Source Schema dropdown menu will be populated with all schemas within the selected database.
  • Source Schema: A dropdown menu to select the schema that contains the table to visualize. Again, making a selection here initiates a script to populate the Source Table dropdown menu with all tables within the selected schema.
  • Source Table: A dropdown menu to select the name of the table to visualize.
  • Get Table: Once the source database, schema and table have been selected, click this button to retrieve the column names of the selected table.

A short time after clicking the Get Table button a second toolbar should appear:

  • X axis: A dropdown menu to select the column in the data table that contains the values to be plotted along the X axis.
  • Y axis: A dropdown menu to select the column in the data table that contains the values to be plotted along the Y axis.
  • #Clusters: A dropdown menu to select the column in the data table that contains the cluster labels. If no column is selected, the bubbles will have a default blue/purple color
  • Labels: A dropdown menu to select the column in the data table that contain the node labels.
  • Importance: A dropdown menu to select the column in the data table whose values define the size of the bubbles.
  • Draw Graph: Once you have selected the X, Y and Labels variables, click this button to retrieve the necessary data and to start drawing the graph. The #Clusters and Importance variables are optional.

Data table

To get the correct visualization, the data table needs to be organized as follows:

The graph will plot a bubble for each row in the table. You are responsible for removing duplicate rows. You will need at least three columns for the X, Y and Labels menus. The table can contain additional columns, the toolbar will let you select the columns that are relevant to the plot.

Known issues

Add a way to test new cases, exclude other test cases from the X nearest nodes evaluation.

Here we show an example of a typical use case and what the resulting plot should look like. In this example we will plot a synthetic data set of movies, arbitrarily assigned to one of three clusters. The data can be found either on Platform (schema.table = pcooman.movies_tsne on redshift-general).

Sample plot

First, we only select the X, Y and Labels variables, selecting the 'x', 'y' and 'movie_name' data table columns respectively:

In the top left corner, there are three icons that allow us to manipulate and explore the plot. Clicking the magnifying glass activates the zoom function. The cursor will change to a cross-hair to indicate this change. Drawing a rectangle will zoom into that area. Double-clicking will zoom out and pan back to the original view. cicking the second icon activates the panning function. The cursor will change to four-pointed arrows to indicate this change. Clicking and dragging will translate the plot. To reset the view, select the zoom icon and double click on the graph. The third icon activates the selector mode. When in this mode, clicking on a node will color it red and mark it as being selected. Later on, we will export the nodes that are nearest to these selected nodes.

The Select by and Select menus offer a more targeted way to highlight and select nodes. The Select by menu offers 3 options: 'class', 'label' and 'importance'. Selecting one will populate the Select dropdown menu options. For now, since we have only selected X, Y and Labels features, we can only select based on label:

Selecting 'label' will populate the Select menu with the movie names (in alphabetical order).

Selecting a label (e.g. 'The Angry Birds Movie') causes the corresponding node to become selected (colored red):

Next, we select 'class_3' from the #Clusters drop down menu and 'true_gross_revenue' from the Importance dropdown menu. Click on 'Draw Graph' to query the data and re-draw the graph. Now, the nodes should have different colors based on their cluster memberships and different size based on their gross revenue:

Hovering over a node should bring up a tooltip, listing the label, cluster membership and importance (depending on whether or not they were provided):

The final set of inputs, located below the graph, allow us to specify an output table and how many nodes should be included. For each node, we will export the 5 nearest nodes. Note that this can include other selected nodes.

The output table will contain a column with the labels of the selected nodes and all information for the 5 nearest nodes:
